## フォルダ構造

```
.ai-team/
├── CLAUDE.md            ← このファイル
└── secretary/
    ├── CLAUDE.md
    ├── inbox/           ← 未分類の入力
    ├── tasks/           ← 日次タスク（YYYY-MM-DD.md）
    └── memos/           ← 決定事項・学び・アイデア
```

## 運用ルール

1. AI秘書が常に窓口 — ユーザーはAIメンバーと直接話さない
2. 同日は1ファイル — 同じ日付ファイルへは追記のみ
3. 決定事項は自動記録 — 依頼されなくても memos/ に保存
4. 日付を必ず確認 — ファイル操作前に現在の日付を確認する
5. 上書き禁止 — 既存ファイルへは追記のみ

## テンプレート変数の対応表

| 変数 | 対応するオンボーディング質問 |
|-----|--------------------------|
| `{{CREATED_DATE}}` | 自動生成（現在の日付） |
| `{{ROLE_AND_WORK}}` | Q1 の回答 |
| `{{GOALS_AND_CHALLENGES}}` | Q2 の回答 |
| `{{TOOLS}}` | Q3 の回答（スキップした場合は「未設定」） |
| `{{ADDITIONAL_MEMBERS}}` | 追加されたAIメンバーの行（初期は空） |
| `{{PERSONALIZATION_NOTES}}` | Q1+Q2 から読み取った特記事項 |

## ADDITIONAL_MEMBERS の行フォーマット

AIメンバーが追加されるたびに以下の行を追記します：

```
| プロジェクト管理AI | プロジェクト・進捗・課題管理 | project/ | 稼働中 |
| リサーチAI | 市場調査・情報収集・比較分析 | research/ | 稼働中 |
| ドキュメントAI | 提案書・報告書・議事録作成 | docs/ | 稼働中 |
| エンジニアリングAI | 技術設計・コードレビュー・デバッグ | engineering/ | 稼働中 |
| データ分析AI | 数値分析・KPI管理・レポート | data/ | 稼働中 |
| クリエイティブAI | デザイン・ビジュアル・UI/UX | creative/ | 稼働中 |
| マーケティングAI | コンテンツ・SNS・キャンペーン | marketing/ | 稼働中 |
| 業務改善AI | ワークフロー改善・自動化提案 | process/ | 稼働中 |
```

非アクティブにする場合はステータスを「一時停止中」に変更します。
