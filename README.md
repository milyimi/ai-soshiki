# AI組織 — AIチーム編成・運用プラグイン

AI秘書を窓口に、専門AIメンバーを配置・活用できるチーム運用スキルです。

## 特徴

- **3ステップでスタート** — 3つの質問に答えるだけでAIチームが立ち上がる
- **AI秘書が窓口** — タスク管理・相談・壁打ちはAI秘書が一手に引き受ける
- **チームは自然に育つ** — 同じ領域の依頼が増えると専門AIメンバーの追加を提案
- **いつでも再編できる** — メンバーの追加・停止・カスタム作成が自在
- 「会社」「社長」などの言葉を使わない設計

## インストール

VS Code の Claude Code 拡張機能の場合：

1. VS Code 左メニューの Claude アイコンを開き、新しいセッションを開始
2. `/plugin` コマンドを実行
3. **Marketplaces** タブで `milyimi/ai-soshiki` を追加し、**Plugins** タブに移動
4. `soshiki` の **Install** をクリック → **Install for you** をクリック
5. VS Code を再起動

## 使い方

1. プロジェクトフォルダで `/soshiki` を実行
2. AI秘書が3つの質問をする（役割・目標・ツール）
3. `.ai-team/` フォルダが作成され、チームが稼働開始

```
/soshiki
```

## AIチームメンバー一覧

| メンバー | 担当領域 |
|---------|---------|
| AI秘書 | コーディネーション・タスク管理・相談（常設） |
| プロジェクト管理AI | プロジェクト・進捗・課題管理 |
| リサーチAI | 市場調査・情報収集・比較分析 |
| ドキュメントAI | 提案書・報告書・議事録作成 |
| エンジニアリングAI | 技術設計・コードレビュー・デバッグ |
| データ分析AI | 数値分析・KPI管理・レポート |
| クリエイティブAI | デザイン・ビジュアル・UI/UX |
| マーケティングAI | コンテンツ・SNS・キャンペーン |
| 業務改善AI | ワークフロー改善・自動化提案 |
| カスタムAI | 独自の役割を自由に定義 |

## フォルダ構造（稼働後）

```
.ai-team/
├── CLAUDE.md              ← チーム設定・メンバー一覧
└── secretary/             ← AI秘書（常設）
    ├── CLAUDE.md
    ├── inbox/             ← 受信トレイ
    ├── tasks/             ← 日次タスク（YYYY-MM-DD.md）
    └── memos/             ← 決定事項・学び・アイデア
```

AIメンバー追加後：
```
.ai-team/
├── CLAUDE.md
├── secretary/
├── research/              ← リサーチAI
│   ├── CLAUDE.md
│   ├── topics/
│   └── reports/
└── project/               ← プロジェクト管理AI
    ├── CLAUDE.md
    ├── plans/
    └── issues/
```

## チーム編成コマンド例

```
「リサーチAIを追加して」
「チームを見せて」
「プロジェクト管理AIを一時停止して」
「チームを再編して」
「データ分析担当を配置して」
「新しい担当を作りたい」（カスタムAIメンバー）
```
